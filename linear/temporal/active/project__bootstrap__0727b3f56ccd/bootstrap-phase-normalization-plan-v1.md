# Bootstrap Phase Normalization Plan v1

Status: Active normalization baseline  
Owner: CJ (final authority), Codex (implementation)  
Source issue: [BIT-83 — Normalize Phase 4/5 milestone scope and progress to reflect actual AI-agent reality](https://linear.app/bitpod-app/issue/BIT-83/normalize-phase-45-milestone-scope-and-progress-to-reflect-actual-ai)

## Purpose

Normalize the `Bootstrap: Org Foundation & Migration` project so milestone progress reflects operational truth instead of narrow doc/spec completion, then use that normalized structure as the default priority spine for autopilot work.

## Core Corrections

- Phase 4 is the true final bootstrap milestone: AI/AI-agent team operational in practice.
- Phase 5 is post-foundation hardening and polish.
- Phase 6 is not part of the bootstrap project. It should begin as separate feature-delivery projects after the foundation is truly ready.

## Interpretation Rules

### Phase 1 — Critical Identity + Access Bootstrap

Historical closeout milestone for identity, secrets, org setup, migration bootstrap, and immediate safety baseline.

### Phase 2 — Integration Parity + Runtime Migration

Transport/runtime parity milestone. Treat as functionally complete only when residual acceptance gates are cleared. Any work still requiring real Discord validation or equivalent live parity proof remains a residual Phase 2 acceptance gate.

### Phase 3 — Retroactive Cleanup + Hardening

Cleanup/hardening milestone. Includes personal-account lockdown, local scope hardening, backup/legacy retirement, drift checks, and historical normalization.

### Phase 4 — AI Agent Team Orchestration + Specialization

True final bootstrap milestone. Not complete when docs/specs exist. Complete only when the operating AI team exists in practice:

- Taylor orchestrator working as the real coordination brain
- specialist lanes defined and actually used
- QA path exists as a real operating lane, even if interim
- session/chat flow works in a real environment
- durable memory/decision/artifact flow is happening through the operational stack

### Phase 5 — Governance + Memory + Eval Hardening

Post-foundation hardening and polish. Baseline governance/memory/eval artifacts may be complete, but operational hardening is not complete until reviewer routing, QA authority maturity, checkpoint protocol in practice, and production-grade traceability are all working.

## Why Prior Progress Was Misleading

The earlier milestone percentages were being driven by baseline artifact completion, not by live operational proof.

Examples:

- [BIT-66 — Agent runtime portability plan: Cloudflare-first execution with platform-decoupled memory/integrations](https://linear.app/bitpod-app/issue/BIT-66/agent-runtime-portability-plan-cloudflare-first-execution-with) is a valid completed plan artifact, but does not prove an AI team exists in practice.
- [BIT-67 — Taylor runtime core hardening: orchestration loop + agent/tool registry v1](https://linear.app/bitpod-app/issue/BIT-67/taylor-runtime-core-hardening-orchestration-loop-agenttool-registry-v1), [BIT-68 — Memory stewardship service v1: proposal/approval flow + contradiction scan](https://linear.app/bitpod-app/issue/BIT-68/memory-stewardship-service-v1-proposalapproval-flow-contradiction-scan), [BIT-69 — Eval + regression gate framework v1 for specialist outputs](https://linear.app/bitpod-app/issue/BIT-69/eval-regression-gate-framework-v1-for-specialist-outputs), [BIT-70 — Governance policy engine v1: high-impact approval thresholds + audit trails](https://linear.app/bitpod-app/issue/BIT-70/governance-policy-engine-v1-high-impact-approval-thresholds-audit), [BIT-71 — Communication surface portability contract v1: Discord baseline + dashboard-ready adapters](https://linear.app/bitpod-app/issue/BIT-71/communication-surface-portability-contract-v1-discord-baseline), and [BIT-73 — Wire governance + durable memory into Taylor runtime service v1](https://linear.app/bitpod-app/issue/BIT-73/wire-governance-durable-memory-into-taylor-runtime-service-v1) are completed baseline groundwork, not proof that Phase 5 is operationally complete.
