# Taylor01 North Star v1

Status: Active
Primary issue: [BIT-100 — Define AI-agent portability boundary and repo extraction strategy](https://linear.app/bitpod-app/issue/BIT-100/define-ai-agent-portability-boundary-and-repo-extraction-strategy)
Project: [Taylor01](https://linear.app/bitpod-app/project/taylor01-b51442062c45)

## Purpose

Define Taylor01 as a co-equal product to BitPod App, with higher long-term strategic importance for CJ's professional stack.

## Core framing

- Taylor01 is CJ's portable PM + AI team operating system.
- Taylor01 includes more than agents:
  - roles
  - workflow
  - workspace policy
  - trust boundaries
  - certainty rules
  - adapter contracts
  - operating norms that travel with CJ
- BitPod App is the first serious proving-ground product built with Taylor01.
- BitPod does not own Taylor01 architecture.
- BitPod completion is not required to validate Taylor01 as a useful product.

## Portfolio model

Operate with two simultaneous north stars:

1. BitPod App
   - the product being built
2. Taylor01
   - the reusable system used to build BitPod and future products

## Control rule

Taylor01 portability concerns may legitimately slow or redirect BitPod work when the alternative would create hard long-term coupling.

That is intentional, not drift.

## Near-term stance

- Keep implementation in-repo for now.
- Define the future extraction boundary now.
- Use BitPod as proof-of-concept without letting BitPod-specific assumptions quietly become Taylor01 defaults.
- Treat Taylor01 as highly experimental and evolving quickly with the AI landscape.
- Optimize for correct bucket placement and explicit temporary coupling, not for pretending the first structure will be final.
