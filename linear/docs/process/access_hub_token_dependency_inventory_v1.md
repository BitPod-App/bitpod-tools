# T01 Access Hub: Token Dependency Inventory v1

Status: Active reference
Owner: Taylor01 / CJ
Related: BIT-333, BIT-329 (T01 Access Hub v1), BIT-511 (OpenClaw retirement)
Last updated: 2026-05-29
Scope: Inventory of all places an Access Hub or dashboard auth token is expected; decision on continuity vs rotation

## Summary

The original "dashboard URL token" was the OpenClaw gateway's local auth mechanism: a bearer token embedded in a URL fragment (`http://127.0.0.1:PORT/#token=XYZ`). OpenClaw has been retired as of 2026-05-28 (BIT-511 Done).

**The token continuity vs rotation decision is resolved**: the old OpenClaw URL token is obsolete. The replacement model (Hermes + Tailscale Serve) does not use a URL token. The canonical access path is Tailnet-private (`https://mini-01.tail1c36c1.ts.net/`), which relies on Tailscale network authentication, not a URL bearer token.

No new token rotation decision is needed for the Access Hub; the model has changed.

## Old Token Model (Retired)

**Format**: `http://127.0.0.1:<PORT>/#token=<TOKEN>` (OpenClaw gateway)

**What it did**: Authenticated local access to the OpenClaw dashboard on localhost. The token was generated by the OpenClaw runtime and rotated on restart.

**Where it appeared:**
- OpenClaw gateway startup output (printed to stdout)
- `openclaw dashboard` CLI command output
- `TAYLOR01_CONTROL_DASHBOARD_URL` env var value (when configured)
- MacBook bookmarks to the OpenClaw local dashboard URL
- Any cached URLs in the Access Hub `dashboard_url` field

**Status**: **Retired.** No OpenClaw processes remain on mini-01.

## Current Token Model (Active)

**Model**: Tailscale Serve + Tailnet-private access

**Routes** (as of 2026-05-28):
| Route | Destination | Auth |
| -- | -- | -- |
| `https://mini-01.tail1c36c1.ts.net/` | Hermes Dashboard :9121 | Tailscale node identity |
| `https://mini-01.tail1c36c1.ts.net/prompt` | PCC :18790 | Tailscale node identity |

**Auth model**: Access is granted by being on the private Tailnet. No URL token is embedded. No rotation cadence required at the application layer.

This cleanup is tracked as a follow-up to BIT-333, not part of this planning doc.

## Token Continuity Decision

**Decision**: No restoration or rotation decision needed.

The original OpenClaw token cannot be "restored" because OpenClaw is retired. The new model (Tailscale Serve) does not use a URL token at all. Operator bookmarks and saved URLs that contained the old OpenClaw `#token=` format are obsolete and should be replaced with the Tailscale Serve URLs above.

**Operator action required**:
- Update MacBook browser bookmarks from `http://127.0.0.1:PORT/#token=...` to `https://mini-01.tail1c36c1.ts.net/`
- Update any saved Access Hub `TAYLOR01_CONTROL_DASHBOARD_URL` env var if it referenced an old OpenClaw URL

## Honcho Broker Tokens (Separate from Dashboard Tokens)

The Access Hub also manages Honcho broker tokens for per-agent authorization. These are separate from the dashboard URL token.

Current status:
- `HONCHO_API_KEY` — loaded from env (sourced from Hermes profile for the Access Hub process)
- `TAYLOR01_CONTROL_HONCHO_BROKER_TOKENS` — JSON map of `{"taylor01": "...", "vera": "..."}` broker tokens

These are active and not part of the OpenClaw token problem. No rotation is needed for BIT-333.
