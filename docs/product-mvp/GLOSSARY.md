# BitPod Glossary (MVP)
*Short, user-friendly definitions. For full vocabulary rules, see `TERMINOLOGY_LIBRARY.md`.*

## Core
- **Baseline:** the default bitregime-core configuration/output. Always runs.
- **Run:** one execution at a cutoff producing reports + receipts.
- **Cutoff:** the time boundary for what information is allowed into a run.

## Tower model
- **Tower:** where Deck broadcasts are ranked by performance (SNR).
- **Deck:** your workspace/configuration that produces broadcasts and earns track record.
- **Control Panel:** where you tune the Deck (faders, toggles) and preview outputs.
- **Antenna Array:** podcasts used to detect emerging narratives/events.
- **Channel Conditions:** Clear / Choppy / Signal Disruption (interference/volatility overlay).

## States
- **LIVE:** broadcasting ON; track record accrues in blocks.
- **RECALIBRATING:** temporary Off-Air; blocks do not accrue.
- **OFF-AIR:** not broadcasting; blocks do not accrue.

## Performance
- **SNR:** the main performance score.
- **Track record (blocks):** earned LIVE history measured in blocks.
- **Deviation from Baseline:** how far your Deck departs from default.
- **Personalization Delta:** what changed vs Baseline for a run.

## Sources
- **Chain Truth Sources:** on-chain reality (hard constraints).
- **Fiat Authority Sources:** official sources that move markets (even if contested).
- **Independent Reality Checks:** cross-check datasets (e.g., Trueflation alongside CPI).
