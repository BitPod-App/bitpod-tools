# Terminology Library (MVP Canon)
*Preferred term → 1-line definition → allowed alts → what to avoid.*

## 🗼 Tower / Radio language

| Preferred | Definition (1 line) | Alts | Avoid |
|---|---|---|---|
| Tower | Public ranking surface for Deck broadcasts. | Leaderboard (internal) | “Top vibes” framing |
| Broadcast | Publishing a forward-scored output. | Go LIVE | “Pin” (user-facing) |
| LIVE | Broadcasting ON; blocks accrue. | On-Air | “Pinned” (user-facing) |
| RECALIBRATING | Temporarily Off-Air; blocks do not accrue. | Off-Air (temp) | “Punished” tone |
| OFF-AIR | Not broadcasting; not rank-eligible. | Inactive | “Unpinned” (user-facing) |
| Antenna Array | Podcast feeds used for detection. | Antennae | “Influencers” as authority |
| Channel Conditions | Clear / Choppy / Signal Disruption overlay. | Channel state | “Signal” used negatively |
| Interference | Volatility framed as channel disruption. | Disruption | Moralized volatility |
| gain / fade | Small rank change. | rise / fall | “dip” (default UI) |
| spike / drop | Large rank change quickly. | surge / slide | “crash” (overused) |
| Signal | Useful information (always positive). | — | “signal” as insult |
| Noise floor | Baseline instability of performance. | — | Confusing noise with “bad news” |

## ⛏️ Timechain / Bitcoin language

| Preferred | Definition (1 line) | Alts | Avoid |
|---|---|---|---|
| Block height | Canonical time reference. | Block time | Wall-clock as primary |
| Track record (blocks) | Earned LIVE history in blocks. | Broadcast streak | “Block record” (ambiguous) |
| Off-Air blocks | Blocks not accrued while RECALIBRATING/OFF-AIR. | Missed blocks | Punitive tone |
| Chain Truth Sources | On-chain reality + transparent derivations. | On-chain truth | Calling fiat “truth” |
